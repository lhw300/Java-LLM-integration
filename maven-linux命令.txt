在 Linux 环境下，尤其是当你遇到 GLIBC_2.29' not found 这种底层库版本不匹配的问题时，升级 glibc 是一个非常敏感且高风险的操作，因为它是系统的核心组件。

如果你的环境是 WSL（从你的路径名 call@DESKTOP-EVBG3PA 来看极大概率是），最稳妥且推荐的方法是直接更换一个更高版本的 Linux 发行版（如 Ubuntu 22.04），而不是手动升级现有的 glibc。


sudo apt update
sudo apt upgrade libstdc++6 libc6
ldd --version

如果你使用的是旧版 Ubuntu（如 18.04），它默认的 glibc 版本确实不支持 PyTorch 2.2.2。你可以通过 Microsoft Store 或命令行安装 Ubuntu 22.04 LTS，它自带的 glibc 版本通常为 2.35，完全能满足需求。

你现在的系统环境 (CentOS 8 + GLIBC 2.28) 确实无法满足本地部署 PyTorch 2.2.2 的硬件/系统底层要求。

如果你想在 Linux 上跑本地模型：请安装 CentOS Stream 9 或 Ubuntu 22.04。
